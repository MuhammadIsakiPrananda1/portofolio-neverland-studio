# Development Guide

Step-by-step instructions for running Neverland Studio locally.

---

## Prerequisites

| Tool | Minimum version |
|------|----------------|
| Node.js | 20 LTS |
| npm | 10 |
| PHP | 8.2 |
| Composer | 2.x |
| MySQL | 8.0 |
| Git | 2.x |

---

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/MuhammadIsakiPrananda1/portofolio-neverland-studio.git
cd portofolio-neverland-studio

# 2. Install frontend dependencies
npm install

# 3. Configure frontend environment
cp .env.example .env
# Edit .env — at minimum set VITE_API_URL

# 4. Install backend dependencies
cd backend
composer install
cp .env.example .env
php artisan key:generate

# 5. Run database migrations (and optional seeders)
php artisan migrate --seed

# 6. Start backend dev server
php artisan serve --port=8001

# 7. In a new terminal — start frontend dev server
cd ..
npm run dev
```

The SPA is now available at **http://localhost:5173** and the API at **http://localhost:8001**.

## Environment Variables

### Frontend (`.env`)

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_API_URL` | Backend base URL | `http://localhost:8001` |
| `VITE_API_VERSION` | API version prefix | `/api/v1` |
| `VITE_PUSHER_APP_KEY` | Pusher app key for real-time | — |
| `VITE_PUSHER_HOST` | Pusher / Soketi host | `localhost` |
| `VITE_PUSHER_PORT` | Pusher / Soketi port | `6001` |
| `VITE_PUSHER_SCHEME` | `http` or `https` | `http` |

### Backend (`backend/.env`)

See `backend/.env.example` for the full list. Key variables:

| Variable | Description |
|----------|-------------|
| `APP_KEY` | Generated by `php artisan key:generate` |
| `DB_*` | MySQL connection settings |
| `BROADCAST_DRIVER` | `pusher` or `log` (dev) |
| `PUSHER_APP_*` | Pusher credentials |

## Available Scripts

### Frontend

```bash
npm run dev       # Start Vite dev server (HMR)
npm run build     # TypeScript check + production build
npm run preview   # Preview the production build locally
npm run lint      # ESLint check
```

### Backend

```bash
php artisan serve          # Dev HTTP server
php artisan test           # Run PHPUnit test suite
php artisan migrate        # Run pending migrations
php artisan migrate:fresh --seed  # Reset DB + seed
php artisan queue:work     # Process queued jobs
```

### Utility scripts

```bash
./scripts/setup-port8080.sh   # Bind dev server to port 8080
```

## Development Tips

- **Path aliases** — The frontend uses Vite aliases (`@components`, `@pages`, `@hooks`, etc.). See `vite.config.ts` for the full list.
- **Atomic Design** — Place new UI elements in the correct layer (`atoms` → `molecules` → `organisms`). Each component folder should have a component file **and** a barrel `index.ts`.
- **Lazy loading** — All page-level components must be lazy-loaded through `src/config/routes.config.tsx`. Do not import pages directly in `App.tsx`.
- **Type safety** — All new code must be in `.tsx` / `.ts`. Avoid `any`; add missing types to `src/types/index.ts`.
- **Real-time** — Dashboard real-time features use Laravel Echo. Initialize it via `src/utils/echo.ts`.
